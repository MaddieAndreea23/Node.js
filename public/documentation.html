Documentation

myFlix consists in the server­side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

URL HTTP Method Request body data format Response body data format
Gets the list of data about ALL movies /movies GET None A JSON object holding list with data about all the Movies
Gets the data about a single movie, by title /movies/[Title] GET None A JSON object holding data about a single movie, containing a title, description, genre, director and image. Example:

{ "Genre": { "Name": "Adventure", "Description": Adventure movies are usually exciting stories, with new experiences or exotic locales, very similar to or often paired with the action film genre." }, "Director": { "Name": "Peter Jackson", "Bio": "Sir Peter Robert Jackson is a New Zealand film director, screenwriter, and film producer. He is best known as the director, writer, and producer of the Lord of the Rings trilogy (2001–03) and the Hobbit trilogy (2012–14), both of which are adapted from the novels of the same name by J. R. R. Tolkien." } }

Gets the list of data about ALL genres /genres GET None A JSON object holding a list with data about all the Genres
Gets the data about a single genre, by name /genres/[Name] GET None A JSON object holding data about a movie genre. Example:

{ "Name": "Action", "Description": "Action movies normally include high energy, big-budget physical stunts and chases, battles, fights, escapes, destructive crises - all designed for pure audience escapism." }

Gets the list data about ALL directors /directors GET None A JSON object holding a list with data about all the Directors
Gets the data about a single director, by name /directors/[Name] GET None A JSON object holding data about a director. Example:

{ "Name": "Ted Demme", "Bio": "Edward Kern 'Ted' Demme was an American director, producer, and actor.", "Birth": "1963", "Death": "2002" }

Gets the list data about ALL Users /users GET None A JSON object holding a list with data about all the Users
Gets the data about a single user, by name /users/[Name] GET None A JSON object holding data about a user. Example:

{ "Name": "Ben Cohen", "Password": "cohenben143", "Email": ben.cohen@gmail.com", "Birthday": "23-06-1992" }

Add data for a new user /users POST A JSON object holding data about the user to add, structured like:

{ "Username": "Laura", "Password": "Laura12345", "Email":"laura@hotmail.com", "Birthday": "11-11-1993" }

Deletes a user from our list by id /users/[Id] DELETE None A message indicating thar user was removed from the list of users
